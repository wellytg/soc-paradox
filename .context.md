# .context.md: soc-paradox

## 1. Domain Context
The `soc-paradox` repository is the landing page and research data repository for the **SOC Strategic Alignment Research Day**. It provides a central portal for participants to access the live simulation and view preliminary research data regarding LLM agreement in Security Operations Centers (SOC).

## 2. Core Architecture
- **Frontend:** A simple, high-impact landing page (`index.html`) using Tailwind CSS for styling.
- **Data Presentation:** Standalone HTML dashboards (`Preliminary full matrix/SOC_T1_Agreement_Dashboard.html`) that embed data for offline/portable viewing.
- **Deployment:** Continuous Deployment (CD) to GitHub Pages via a GitHub Actions workflow.

## 3. Tech Stack
- **HTML/CSS:** Standard HTML5 with Tailwind CSS (CDN-based).
- **CI/CD:** GitHub Actions (`deploy.yml`).
- **Data:** JSON-formatted research data (`comparison_web.json`).
- **Assets:** PDF research posters.

## 4. Key Workflows
- **Deployment:** Any push to the `main` branch triggers an automatic rebuild and deployment to GitHub Pages.
- **Research Access:** Users visit the landing page to launch the external simulation hosted at `wellytg.github.io/soc-paradox-sim`.

## 5. Domain Constraints
- **Static Only:** The repository is designed for static content deployment.
- **External Dependencies:** The landing page points to an external simulation environment.
